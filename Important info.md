# Important Info — Downloads Intelligence

- **Build type:** Automation (run with Claude in this folder, on-demand or scheduled — not a plugin, not a hosted app).
- **This is the user's second AI automation project** (first referenced as a "PDF/Document Intelligence" project — not accessible in this workspace, so this project's architecture was designed fresh from the shared CLAUDE.md template rather than copied).
- **No code has been written yet** — this session was design-only: folder structure, pipeline spec, naming conventions, logs/metadata schema.
- **Examples pending:** user plans to drop 3–5 real Downloads example files into `Build-out/00 Pre-build context & examples/` — tools/templates in the spec should be revisited once those land.
- **Non-negotiables from the brief:** never permanently delete anything; every action must be reversible; duplicates/superseded versions get archived, not deleted.
- **This vault ≠ the real Downloads folder.** This folder is the build/planning workspace; the automation will eventually act on the user's actual OS Downloads folder, connected separately when it runs.
- **Design review (2026-07-05) locked in:** `Registry/` → renamed in spirit to `Database/` (split into `Metadata/`/`FileIndex/`/`History/`; old `Registry/` folder kept only as a dead stub — can't delete files in this workspace). Added `Rules/` (modular business rules, separate from the spec), `Tests/` (5 datasets, prioritize Small Batch → Mixed Downloads → Duplicate Files → Corrupted Files → Large Batch), split `Logs/` (machine) from `Reports/` (human-readable), added `Learning/` (passive correction capture, not acted on yet).
- **Destination taxonomy simplified** to 8 flat folders for v1: Documents, Finance, Images (+ Screenshots subfolder), Videos, Audio, Applications, Archives, Unknown. Documents/ subfolders (Contracts, Resumes, Certificates, Legal, Manuals) are documented but NOT pre-created — add only once volume justifies it.
- **Confidence scoring is now a fixed, auditable deduction system** (start at 100, named deductions, hard floors) — see `Rules/Confidence Rules.md`. Never an arbitrary AI-reported percentage.
- **Execution modes:** Manual is the v1 default and what gets built/tested first. Scheduled is cheap to add after (same logic, timer-triggered). Watch Folder (real-time background daemon) is intentionally deferred — bigger lift, not worth building before the core pipeline is proven.
- **Multi-source extensibility:** the pipeline is designed around a "Source" concept (v1 has exactly one Source: Downloads) so Desktop/Google Drive/OneDrive/Dropbox can be added later without redesigning steps 02–08.
- **Second design review (2026-07-05) locked in:** `review_required` files are left in their original location and flagged (no dedicated folder — fixed a contradiction between the 06 and 07 docs); added a generic `Document` classification category (mapped to `Documents/`) so ordinary readable files stop falling into `Unknown`; moved naming templates into `Rules/Naming Rules.md` for consistency with the other business rules; added `README.md` (primary human-facing doc — `CLAUDE.md` now just points to it), `ROADMAP.md` (consolidates all "future version" mentions), `CHANGELOG.md` (replaces 6 separate per-file change-log footers), and `Samples/` (canonical examples: Invoices/Images/Videos/Documents — distinct from `Tests/`, which holds scenario-based validation datasets, no overlap). Full detail: `CHANGELOG.md`.
- **Architecture was frozen** after the second design review, then implementation began. **Implementation-time restructure (2026-07-05):** added `src/` for actual code (standard Python package layout, one module per pipeline step), moved `Logs/`+`Reports/` under a new `Runtime/` folder (operational output, distinct from `Database/`), and moved `Learning/` under `Database/` (it's persistent structured data like Metadata/FileIndex/History). Old top-level `Logs/`, `Reports/`, `Learning/` folders are now deprecated stubs, same treatment as `Registry/`. Full detail: `CHANGELOG.md`. Note: bullet above (design review) describes the pre-restructure names — `Logs/`/`Reports/`/`Learning/` there now mean `Runtime/Logs/`, `Runtime/Reports/`, `Database/Learning/`.
